## 1. 项目概述

### 1.1 项目背景

学术调研与综述写作是科研工作的核心环节，但传统流程存在显著痛点：

1. 论文检索效率低：人工筛选关键词、跨平台检索耗时久，易遗漏关键文献
2. 阅读成本高：单篇综述需精读数十至上百篇文献，信息提取与整理周期长
3. 跨文献分析困难：多文献方法论对比、技术演进梳理、全局趋势总结依赖人工经验
4. 综述写作周期长：从调研到成稿通常需要数天至数周，效率低下


### 1.2 项目目标

基于 LangGraph 框架构建具备状态管理、循环与条件分支能力的 Multi-Agent 系统，协调多智能体自动完成「搜索 - 阅读 - 分析 - 写作 - 报告」全流程，实现学术调研与综述生成的全自动化，将传统人工调研周期压缩至 1 小时以内，大幅提升科研效率。

- 效率提升：将数天至数周的人工调研流程，压缩至 1 小时内完成，实现效率数量级提升
- 标准化：统一调研流程与信息提取规范，降低人工主观偏差
- 智能化：基于大语言模型与向量检索技术，实现文献深度语义分析与结构化写作
- 可扩展性：模块化 Agent 架构支持快速迭代与功能扩展，适配不同学科调研需求


## 2. 系统架构设计

### 2.1 整体架构

系统基于LangGraph框架实现 Multi-Agent 协同，核心分为 5 大智能体模块，各模块通过状态机实现任务流转、循环校验与条件分支，整体架构如下：

```text
用户需求输入 → 搜索智能体 → 阅读智能体 → 分析智能体 → 写作智能体 → 报告生成智能体 → 最终调研报告输出
```

- 核心框架：LangGraph（负责状态管理、任务调度、循环 / 分支控制）
- 大语言模型（LLM）：核心能力支撑，负责检索条件生成、信息提取、分析、写作、格式优化
- 向量数据库：Chroma，存储结构化论文信息，支持语义检索
- 嵌入模型：bge-small-zh-v1.5，用于文本向量化与语义相似度计算
- 工具集成：MCP（Model Context Protocol）工具，用于学术论文检索

### 2.2 模块依赖关系

| 模块 | 上游依赖 | 下游输出 |
| --- | --- | --- |
| 搜索智能体 | 用户调研需求 | 结构化论文元数据列表 |
| 阅读智能体 | 论文元数据列表 | 结构化论文核心信息（存入 Chroma 向量库） |
| 分析智能体 | Chroma 向量库中的论文信息 | 主题分组、深度分析报告、全局洞察报告 |
| 写作智能体 | 分析智能体输出、Chroma 向量库 | 分章节结构化综述草稿 |
| 报告生成智能体 | 分章节草稿 | 符合 Markdown 规范的完整调研报告 |


## 3. 核心模块详细设计

### 3.1 搜索智能体 (Search Agent)

#### 3.1.1 核心职责

根据用户输入的调研需求，自动生成精准的检索条件，完成论文检索与人工审核，输出结构化论文元数据。

#### 3.1.2 核心流程

1. 检索条件生成：调用 LLM，基于用户需求生成结构化检索条件（关键词组合、时间范围、文献类型、期刊范围等）
2. 论文检索：通过 MCP 工具执行检索，支持多关键词组合、时间范围筛选、相关性排序
3. 人工审核节点：将检索结果返回给用户，支持人工增删文献、调整检索条件，确保检索准确性
4. 元数据结构化：对最终确认的文献，提取并结构化存储论文元数据（标题、作者、发表年份、期刊、DOI、摘要、链接等）

#### 3.1.3 输入输出

- 输入：用户调研需求（文本）
- 输出：结构化论文元数据列表（JSON 格式）

### 3.2 阅读智能体 (Reading Agent)

#### 3.2.1 核心职责

基于 LangChain 架构，批量处理论文文本，精准提取核心信息，并向量化存储至 Chroma 向量数据库。

#### 3.2.2 核心流程

1. 论文文本加载：批量加载论文 PDF / 文本内容
2. 核心信息提取：调用 LLM，从论文中精准提取以下结构化字段：
3. 核心研究问题
4. 关键技术方法
5. 实验数据与结果
6. 核心创新与贡献
7. 局限性与未来工作
8. 向量化存储：使用bge-small-zh-v1.5嵌入模型，将结构化信息转换为语义向量，存入 Chroma 向量数据库，支持后续语义检索

3.2.3 输入输出
输入：结构化论文元数据列表、论文原文（PDF / 文本）
输出：Chroma 向量数据库中的结构化论文信息（可检索）
3.3 分析智能体 (Analysis Agent)
3.3.1 核心职责
由 3 个逐层递进的子智能体构成，完成从主题分组到全局洞察的全链路分析，为综述写作提供核心内容支撑。
3.3.2 子智能体设计
3.3.2.1 聚类子智能体 (Clustering Sub-Agent)
核心职责：对论文进行自动主题分组，生成主题描述
核心流程：
从 Chroma 向量库提取论文结构化信息，通过bge-small-zh-v1.5生成语义向量
采用 K-means 聚类算法进行自动主题分组，通过肘部法则确定最佳聚类数量
调用 LLM，为每个聚类生成主题描述、核心关键词、代表性文献
输入：Chroma 向量库中的论文信息
输出：主题分组结果（主题名称、关键词、对应文献列表）
3.3.2.2 深度分析子智能体 (In-depth Analysis Sub-Agent)
核心职责：针对每个主题，完成方法论对比、性能评估与技术演进分析
核心流程：
针对每个主题，从向量库检索对应文献
调用专用 LLM，完成：
方法论横向对比（技术路线、核心思路、优缺点）
性能指标评估（实验数据、效果对比、适用场景）
技术演进脉络梳理（时间线、关键突破、迭代逻辑）
输入：主题分组结果、Chroma 向量库
输出：单主题深度分析报告
3.3.2.3 全局分析子智能体 (Global Analysis Sub-Agent)
核心职责：整合所有主题结果，生成全局洞察报告
核心流程：
汇总所有主题的深度分析结果
调用 LLM，完成全局归纳：
领域整体发展趋势
跨主题技术对比与关联
当前研究热点与前沿方向
领域现存局限性与未来研究方向
输入：所有主题的深度分析报告
输出：全局洞察报告
3.4 写作智能体 (Writing Agent)
3.4.1 核心职责
由 3 个协同子智能体构成，基于分析结果，通过「资料评估 - 内容撰写 - 审核确认」的迭代机制，完成综述各章节写作。
3.4.2 子智能体设计
3.4.2.1 写作主管子智能体 (Writing Manager Sub-Agent)
核心职责：拆解写作任务，生成结构化章节大纲
核心流程：
基于全局洞察报告与主题分析结果，调用 LLM 的规划能力
生成符合学术规范的综述大纲，包含：
摘要、引言、研究背景
分主题技术综述
方法对比与分析
趋势与展望
结论
支持人工调整大纲，确认后进入下一环节
输入：全局洞察报告、主题分析结果
输出：结构化综述章节大纲
3.4.2.2 资料检索子智能体 (Data Retrieval Sub-Agent)
核心职责：根据大纲，从 Chroma 向量库检索对应主题的文献资料
核心流程：
针对大纲每个章节，生成检索关键词
调用 Chroma 向量库的语义检索能力，获取相关论文的结构化信息
按相关性排序，为章节写作提供素材
输入：章节大纲、Chroma 向量库
输出：章节对应文献素材
3.4.2.3 章节写作子智能体 (Chapter Writing Sub-Agent)
核心职责：基于大纲与素材，完成单章节写作
核心流程：
针对每个章节，输入大纲要求与对应文献素材
调用 LLM，完成章节内容撰写，确保：
符合学术写作规范
内容逻辑连贯、论据充分
正确引用文献
进入「审核确认」环节，支持人工修改，迭代优化
输入：章节大纲、对应文献素材
输出：单章节综述草稿
3.4.3 迭代机制
系统采用条件循环与状态判断机制，在「资料评估 - 内容撰写 - 审核确认」中迭代：
资料评估：检查素材是否充分，不足则重新检索
内容撰写：生成章节内容
审核确认：人工 / 自动审核内容准确性、逻辑性，不达标则重新撰写
循环直至所有章节达标
3.5 报告生成智能体 (Report Generation Agent)
3.5.1 核心职责
整合所有章节内容，完成逻辑重组、语句连贯与排版优化，生成符合 Markdown 规范的完整调研报告。
3.5.2 核心流程
内容整合：汇总所有章节草稿，按大纲顺序重组
逻辑优化：调用 LLM，优化章节间过渡、语句连贯性，消除内容重复
格式标准化：统一 Markdown 格式（标题层级、列表、引用、公式、参考文献格式）
最终校验：检查内容完整性、学术规范、排版一致性
输出报告：生成可直接使用的 Markdown 格式完整调研报告
3.5.3 输入输出
输入：分章节综述草稿、大纲
输出：符合 Markdown 规范的完整学术调研报告
4. 技术栈与依赖
4.1 核心框架
多智能体调度：LangGraph（状态管理、循环 / 分支控制）
智能体基础架构：AutoGen（AssistantAgent）
大语言模型：支持 OpenAI GPT 系列、通义千问、文心一言等主流 LLM
4.2 向量与检索
向量数据库：Chroma
嵌入模型：bge-small-zh-v1.5（中文语义向量化）
检索工具：MCP (Model Context Protocol) 学术检索工具
4.3 数据处理
聚类算法：K-means（肘部法则确定最佳聚类数）
文本处理：PyPDF2/pdfplumber（PDF 解析）、LangChain（文本处理）

Python 3.10+
内存：≥16GB（向量存储与 LLM 推理）
存储：≥100GB（文献存储与向量库）
网络：可访问学术数据库与 LLM API
